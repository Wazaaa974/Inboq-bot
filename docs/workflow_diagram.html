Automatique
Paiement Stripe
Le client paie (39€ Starter ou 89€ Pro). Stripe envoie un webhook à inboq.nanocorp.app et redirige le client vers la page de succès. Tu reçois l'email + montant + session ID.
⚡ 0 min — automatique
💳
Inboq
Email de bienvenue
Tu envoies l'email de bienvenue au client avec le questionnaire (10 questions) et les instructions pour créer son bot via @BotFather.
Contenu de l'email :
  • Confirmation de l'abonnement
  • Lien / questionnaire à remplir
  • Instructions @BotFather pas à pas
  • Adresse email pour envoyer le token
⏱ ~10 min · Délai cible : < 2h après paiement
📧
Client
Questionnaire + Token Telegram
Le client remplit le questionnaire (services, tarifs, horaires, règles, FAQ…) et crée son bot via @BotFather. Il t'envoie le token par email.
Si pas de réponse : relancer après 48h
⏱ Dépend du client (~10-15 min de son côté)
📝
Inboq
Création du JSON de config
Tu copies client_config_template.json et tu le remplis avec les réponses du questionnaire. Valider que le JSON est syntaxiquement correct.
Fichier créé : clients/{client_id}_config.json
⏱ ~15 min
⚙️
Inboq
Génération du System Prompt
À partir du JSON, tu génères le PROFESSIONAL_PROFILE — la "personnalité" du bot. Ce texte sera collé dans la variable Railway.
Inclut : profil, services, tarifs, dispo, règles, FAQ, ton, qualification
⏱ ~5 min
🧠
Technique
Déploiement Railway
Créer un nouveau projet Railway depuis le repo GitHub Wazaaa974/Inboq-bot. Configurer les 5 variables d'environnement. Vérifier les logs.
Variables à renseigner :
  • TELEGRAM_BOT_TOKEN
  • OPENAI_API_KEY
  • ADMIN_CHAT_ID (mettre 0 provisoirement)
  • PROFESSIONAL_PROFILE
  • WELCOME_MESSAGE
⏱ ~15 min
🚂
Technique
ADMIN_CHAT_ID + Tests
Envoyer /start au bot → récupérer le chat_id → mettre à jour la variable Railway. Puis tester les 4 scénarios clés.
Scénarios de test :
  • /start → message de bienvenue correct ?
  • Question FAQ → bonne réponse ?
  • Demande de RDV → qualification ?
  • Notification admin → reçue en privé ?
⏱ ~15 min
🧪
Inboq
Email go-live 🎉
Envoyer l'email de confirmation au client avec le lien vers son bot (t.me/username_bot). Lui rappeler qu'il peut demander des modifications par email.
⏱ ~5 min · SLA total : 24-48h après paiement
🎉
